# 邻动体适能小程序 回归测试清单

## 1. 说明

本文档用于当前小程序主链路的人工回归测试，重点覆盖：

* 登录与多用户切换
* 首页、课程详情、支付确认、支付结果、拼团详情、我的拼团
* 开团、参团、分享、重复参团拦截
* 未支付返回、支付失败、权限校验、状态同步

建议每次回归前先确认以下事项：

* 当前后端服务已启动：`http://127.0.0.1:8000`
* 小程序已重新编译为最新代码
* `utils/auth.js` 中 mock 登录开关已按需要配置
* 每次切换测试用户前，先在“我的”页执行退出登录
* 如需验证“立即开团 / 去参团 / 确认支付”主链路，先执行 `cd backend && npm run seed:regression-course`

## 2. 测试准备

### 2.1 Mock 用户

当前默认推荐使用固定 mock 身份：

* `seed0326_u02`：当前小程序默认登录身份，可直接验证“立即开团”和“去参团”
* `seed0326_u01`：脚本会把它放进“可直接参团课程”的活跃团里，作为已在团内的种子成员

切换用户方法：

1. 打开 [auth.js](/Users/yun/lindong/miniprogram/utils/auth.js)
2. 修改 `const MOCK_OPEN_ID = 'seed0326_u02'` 为目标用户
3. 保持 `const USE_MOCK_USER = true`
4. 在小程序“我的”页先退出登录，再重新登录
5. 确认当前用户已切换

### 2.2 建议关注的接口

测试时建议同步观察开发者工具 `Network` / 控制台：

* `GET /api/courses`
* `GET /api/courses/:id`
* `GET /api/courses/:id/active-group`
* `POST /api/orders`
* `POST /api/payments/mock-success`
* `GET /api/groups/:id`
* `GET /api/user/groups`

### 2.3 测试数据准备

用于验证主链路时，建议先准备两门脚本维护的课程：

1. 进入后端目录：`cd backend`
2. 执行：`npm run seed:regression-course`
3. 记录输出中的两个 `courseId`

说明：

* 该脚本对应文件为 [create-test-course.js](/Users/yun/lindong/backend/scripts/create-test-course.js)
* 脚本为幂等设计，重复执行会重置这两门课程到可回归状态
* 默认会准备：
  * `"[回归测试] 无活跃拼团课程"`：用于验证“立即开团 -> 确认支付 -> mock success”
  * `"[回归测试] 可直接参团课程"`：用于验证“去参团 -> 确认支付 -> mock success”
* 当前固定 mock 用户 `seed0326_u02` 可直接使用这两门课完成主链路验证

## 4. 回归清单

### 4.1 基础浏览与登录

- [ ] 首次进入小程序时，协议页正常展示；未同意时不能进入首页。
- [ ] 未登录状态可浏览首页课程列表。
- [ ] 未登录状态可进入课程详情页查看课程信息。
- [ ] 未登录状态点击课程详情底部主按钮，会提示前往登录。
- [ ] 在“我的”页可正常登录，登录后头像、昵称、token 正常写入。
- [ ] 切换 `MOCK_USER_ID` 后，退出再登录可切换到新用户，用户态不串号。

### 4.2 首页课程列表

- [ ] 首页首次进入可正常加载课程列表。
- [ ] 顶部定位栏正常展示，定位失败时可回退默认位置。
- [ ] “全部课程 / 最近开课” Tab 切换正常。
- [ ] 下拉刷新正常。
- [ ] 上拉加载更多正常。
- [ ] 有进行中拼团的课程，课程卡片展示“已拼人数”与倒计时。
- [ ] 无进行中拼团的课程，不展示“已拼人数”和倒计时。
- [ ] 从其他页面返回首页后，课程卡片人数能刷新为最新值。

### 4.3 课程详情展示

- [ ] 课程详情页轮播图、课程信息、介绍、拼团规则、教练信息、保险说明正常展示。
- [ ] 有活跃拼团时，课程详情页展示当前拼团进度、已拼人数、倒计时。
- [ ] 无活跃拼团时，课程详情页显示“立即开团”。
- [ ] 当前用户未参团且有活跃拼团时，主按钮显示“去参团”。
- [ ] 当前用户已参团且拼团进行中时，主按钮显示“已参团，去邀请好友”。
- [ ] 点击“已参团，去邀请好友”后，直接拉起微信分享，不先跳拼团详情页。
- [ ] 分享链接打开后落地到课程详情页，并携带 `groupId` 以承接参团流程。

### 4.4 开团成功链路

- [ ] 使用 `seed0326_u02` 进入 `"[回归测试] 无活跃拼团课程"`，主按钮显示“立即开团”。
- [ ] 点击“立即开团”后可进入支付确认页。
- [ ] 支付确认页正常展示订单信息、拼团说明、协议勾选区域。
- [ ] 点击“确认支付”后，订单创建成功并模拟支付成功。
- [ ] 成功进入支付结果页，状态为成功。
- [ ] 点击“查看拼团详情”后进入拼团详情页。
- [ ] 拼团详情页显示进行中状态，人数正确。
- [ ] 返回首页后，课程卡片“已拼人数”同步更新。
- [ ] “我的拼团”页出现该拼团记录，状态为进行中。

- [ ] 使用 `seed0326_u02` 进入 `"[回归测试] 可直接参团课程"`。
- [ ] 课程详情页主按钮显示“去参团”。
- [ ] 点击后可正常进入支付确认页，不应因为拼团详情权限报 `403`。
- [ ] 支付确认页展示的课程信息与当前拼团说明正确。

- [ ] 支付结果页成功，跳转拼团详情后人数 `+1`。
- [ ] 首页课程卡片人数、课程详情页人数、拼团详情页人数保持一致。
- [ ] `seed0326_u02` 的“我的拼团”中出现该记录。

### 4.6 参团未支付返回

### 4.7 支付失败链路

- [ ] 在支付确认页点击“模拟支付失败”可进入失败结果页。
- [ ] 失败结果页展示失败文案，并支持重新支付或返回首页。
- [ ] 支付失败后，课程详情页人数不应增加。
- [ ] 支付失败后，首页课程卡片人数不应增加。
- [ ] 支付失败后，“我的拼团”中不应新增该记录。

### 4.8 已参团用户再次访问课程

- [ ] 已参团用户从首页进入自己已参团且仍在进行中的课程。
- [ ] 主按钮显示“已参团，去邀请好友”。
- [ ] 点击按钮直接进入分享流程。
- [ ] 分享文案与分享图片正常。
- [ ] 分享打开后，落地页为对应课程详情页，并能继续承接参团。

### 4.9 重复参团与重复开团拦截

- [ ] 已成功参团的用户再次尝试参与同一课程拼团时，会被拦截。
- [ ] 拦截文案为“你已参加过该课程的拼团，不能重复参加”。
- [ ] 被拦截后不会生成假成功订单。
- [ ] 被拦截后不会导致首页、课程详情、拼团详情人数异常增加。

### 4.10 拼团详情权限与按钮逻辑

- [ ] 已参团成员可正常访问拼团详情页。
- [ ] 进行中的拼团中，成员看到底部按钮为“邀请好友”。
- [ ] 进行中的拼团中，未参团用户若允许访问详情，则看到“立即参团”；若当前逻辑不允许访问，则应收到合理拦截提示。
- [ ] 已成团状态下，拼团详情页显示状态文案，不显示可继续参团按钮。
- [ ] 已失败状态下，拼团详情页显示“拼团失败，已退款”相关文案。

### 4.11 我的拼团列表

- [ ] “我的拼团”页可正常加载。
- [ ] 顶部 Tab “全部 / 进行中 / 已成团 / 已失败” 切换正常。
- [ ] 当某个 Tab 下无记录时，仅提示“暂无拼团记录”。
- [ ] 列表项展示课程图、名称、时间、地点、状态正常。
- [ ] 点击列表项可进入对应拼团详情页。

### 4.12 状态同步一致性

- [ ] 首页课程卡片的已拼人数，与课程详情页展示一致。

- [ ] 支付成功后，首页、课程详情、拼团详情、我的拼团都能同步到最新状态。
