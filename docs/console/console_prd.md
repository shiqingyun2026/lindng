## 1. 产品概述



### 1.1 项目背景

运营后台是邻动体适能小程序的内部管理工具，供平台运营人员使用，用于配置课程、管理订单、维护管理员账号。后台所有课程配置数据将直接决定小程序家长端展示的课程信息，因此字段设计需与家长端保持严格一致。



### 1.2 产品定位

一个面向内部运营人员的Web管理后台，用于支撑小程序的日常运营工作，保证课程信息准确、订单处理及时、后台账号安全可控。



### 1.3 核心目标

* 提供课程管理功能，支持课程的新增、编辑、下架，课程字段与家长端展示字段一一对应。

* 提供订单管理功能，支持订单查看、搜索、手动退款，订单字段与家长端订单信息一致。

* 提供账号管理功能，支持后台管理员账号的增删改查。

* 确保后台访问安全，通过登录认证控制权限。

* 建立统一课程状态机，确保课程状态、拼团结果、退款处理在后台与家长端口径一致。

## 3. 详细功能需求



### 3.1 登录认证



#### 3.1.1 登录页

* **页面元素**：

  * 用户名输入框

  * 密码输入框（掩码）

  * 登录按钮

* **交互**：

  * 输入用户名和密码，点击登录，后端验证。

  * 验证成功：跳转至后台首页（概览页），存储token。

  * 验证失败：提示“用户名或密码错误”。

* **规则**：

  * 密码需加密传输（HTTPS），后端使用bcrypt存储和比对。

  * token有效期建议7天，过期需重新登录。



#### 3.1.2 概览页

* **页面定位**：

  * 概览页作为当前后台首页。

  * 页面分为两个板块：
    * 数据概括
    * 异常提示

* **时间范围切换**：

  * 支持以下三个范围：
    * 今日
    * 近 7 天
    * 近 30 天

  * 切换后需刷新概览指标，并展示当前统计范围起止日期。

  * 近 7 天 / 近 30 天口径为“含今日”的连续自然日范围。

* **下钻规则**：

  * 概览卡片支持点击后下钻到对应列表页。

  * 下钻时需自动透传默认筛选条件，并在目标列表页中正确回显筛选值。

  * 当前已对齐的下钻口径包括：
    * “当前拼团中的课程” → 课程列表 `status=拼团中`
    * “当前范围内要上课的课程” → 课程列表按 `start_time` 筛选
    * “当前范围内上架课程” → 课程列表按 `publish_time` 筛选
    * “当前范围内成功成团” → 拼团列表按 `success_time` 筛选
    * “当前范围内参团人数” → 拼团列表按 `joined_at` 口径筛选
    * “当前范围内成团金额” → 订单列表按 `pay_time` 筛选
    * “当前范围内系统自动退款订单数” → 订单列表按 `refund_time` + `系统自动退款` 筛选



### 3.2 账号管理



#### 3.2.1 账号列表页

* **列表展示**：

  * 表格字段：用户名、角色（超级管理员/普通管理员）、状态（启用/停用）、最后登录时间、创建时间、操作（编辑、删除）

  * 支持按用户名、角色、状态筛选

  * 支持分页

* **新增账号**：

  * 弹窗表单：用户名、密码、确认密码、角色（下拉选择）

  * 提交后，后端创建账号，密码加密存储

  * 当前实现中支持密码明文 / 隐藏切换，便于人工核对两次输入是否一致

* **编辑账号**：

  * 弹窗表单：用户名（不可修改）、密码（可留空不修改）、角色

  * 提交后更新信息

  * 当前实现中支持同时更新状态（启用 / 停用）

  * 当前实现中如修改密码，也需再次确认密码

* **删除账号**：

  * 点击删除，二次确认后执行软删除或物理删除

* **权限说明**：

  * 超级管理员：拥有所有权限（可管理账号、课程、订单）

  * 普通管理员：仅可查看/管理课程和订单，不可管理账号

  * 当前前端已按角色隐藏“账号管理”导航入口；普通管理员即使直接访问 `/accounts`，也会被重定向回概览页



#### 3.2.2 账号管理规则

* 至少保留一个超级管理员账号，不可删除。

* 新增账号时密码需符合强度要求（至少6位）。



### 3.3 课程管理



#### 3.3.1 课程列表页

* **列表展示**：

  * 表格字段：课程名称、课程类别、报名截止时间、上课时间、所在区域、详细地点、拼团价、成团人数要求、状态、操作（查看、编辑、查看拼团、下架）

  * 支持按课程名称、课程类别、课程状态、时间范围搜索

  * 时间范围支持按不同时间维度筛选：
    * 上课时间
    * 上架时间
    * 报名截止时间
    * 下架时间

  * 支持分页

* **新建课程**：

  * 按钮跳转至新建表单页

* **编辑课程**：

  * 点击编辑进入编辑页，可修改所有字段

* **下架课程**：

  * 点击下架，二次确认后将下架时间更新为当前时间

  * 到下架时间后课程不再处于“拼团中”

* **详情 / 编辑页补充**：

  * 课程详情页支持只读查看

  * 课程详情页需展示课程类别

  * 课程详情页支持查看当前课程下拼团记录

  * 课程详情页、编辑页支持返回上一级



#### 3.3.2 课程状态规则

课程状态统一由后端根据课程时间、拼团结果自动计算，前端只做展示，不允许手工直接修改状态值。

* **状态枚举**：

  * 待上架
  * 已下架

  * 拼团中

  * 拼团失败

  * 等待上课

  * 上课中

  * 已结课

* **状态流转规则**：

  * 新建课程后默认进入“待上架”，必须配置上架时间，且上架时间必须晚于当前时间。

  * 到达下架时间后，课程状态进入“已下架”，前端仅做展示，不允许继续按拼团中课程参与报名。

  * 到达上架时间后，如课程尚无成功拼团的团，状态自动流转为“拼团中”。

  * 若课程在报名截止时间前已有至少一个成功拼团的团，则课程状态流转为“等待上课”。

  * 在开课时间（含）至结束时间（不含）之间，课程状态为“上课中”。

  * 到达结束时间后，课程状态流转为“已结课”。

  * 若到达报名截止时间仍没有一个成功拼团的团，则课程状态流转为“拼团失败”。

* **拼团失败联动规则**：

  * 课程进入“拼团失败”后，系统需自动将该课程下所有未关闭订单标记为已退款。

  * 系统需同步将该课程仍处于进行中的团标记为失败。

  * 系统需保留 `group_members` 记录，作为“用户曾成功支付并加入过该团”的历史事实。

  * 业务限制“同一用户同一课程只能成功参团一次”时，仅按成功团/成功订单判断，不按失败团成员关系判断。

  * 自动退款属于系统行为，需记录退款时间、退款原因，退款原因统一为“报名截止前未成团，系统自动退款”。



#### 3.3.3 拼团管理

拼团仍是课程下的子实体，但当前后台已提供独立“拼团管理”模块，同时保留课程维度入口。

当前版本范围仅包含 P0、P1，不包含手工改团人数、手工改团状态、手工增删团成员等高风险操作。

* **实体关系**：

  * 一个课程下可以有多个历史拼团。

  * 同一时刻仅允许一个 `active` 状态的拼团。

  * 订单归属于某个拼团，拼团再归属于课程。

* **拼团状态**：

  * `active`：进行中

  * `success`：已成团

  * `failed`：已失败

* **拼团关键规则**：

  * 新拼团创建时，其 `expire_time` 必须统一等于课程报名截止时间。

  * 只有支付成功后，用户才真正写入 `group_members` 并增加 `groups.current_count`。

  * 当 `current_count >= target_count` 时，拼团流转为 `success`。

  * 到达 `expire_time` 仍未成团时，拼团流转为 `failed`。

  * 同一用户同一课程只能成功参团一次；失败团和已退款订单不应拦截其未来再次参加其他课程。

* **P0 功能范围**：

  * 拼团列表页：展示拼团ID、所属课程、团状态、当前人数、成团人数要求、开团人、团截止时间、创建时间。

  * 支持按课程名称 / 拼团ID、拼团状态、时间范围筛选。

  * 当前时间范围支持按不同维度筛选：
    * 创建时间
    * 截止时间
    * 成团时间
    * 入团时间

  * 在课程详情或课程列表中提供“查看拼团”入口，可查看该课程下全部历史拼团。

  * 独立拼团管理模块可直接查看全量拼团，不再局限于课程详情内部。

  * 拼团详情页展示团基础信息、成员信息、关联订单与退款结果。

  * 团状态仅可只读展示，不允许人工直接修改 `active / success / failed`。

  * 拼团详情页成员模块中时间字段展示为“入团时间”，其含义为：
    * 支付成功后写入 `group_members` 的时间

  * 当拼团列表按“入团时间”筛选时，统计口径需与概览“参团人数”一致，即基于 `group_members.joined_at`。

  * 用户端“我的拼团记录”与拼团详情页在成功团场景下，仍按拼团状态归类为“已成团”，但会额外展示课程阶段标识：
    * 课程未结课：显示 `等待上课`
    * 课程已结课：显示 `已结课`
    * 即使课程已结课，当前仍归在“已成团”tab 下，不单独拆出“已结课”tab

* **P1 功能范围**：

  * 页面透出关键拼团规则：团截止时间等于课程报名截止时间；课程成功定义为报名截止前至少一个团成功；失败团保留成员历史记录。

  * 拼团详情页展示关联订单列表，包含订单号、用户、金额、支付状态、退款状态、退款时间、退款原因。

  * 增加异常排查提示，例如：课程已拼团失败但仍存在 `active` 团、团人数与成员数不一致、失败团存在未退款订单。

* **当前不纳入范围**：

  * 手工改团人数

  * 手工改团状态为成功

  * 手工增删 `group_members`

  * 脱离订单直接加人进团



#### 3.3.4 课程表单页

表单字段需与家长端展示字段一一对应，详见第4章“数据字段映射”。



| 字段分组  | 字段名    | 字段类型  | 必填 | 说明                |
| ----- | ------ | ----- | -- | ----------------- |
| 基本信息  | 课程名称   | 文本    | 是  | 显示在课程卡片、详情页       |
|       | 课程类别   | 单选    | 是  | 当前支持“体适能 / 跳绳” |
|       | 封面图    | 图片    | 是  | 课程列表卡片封面          |
|       | 适用年龄   | 单选    | 是  | 3-6岁 / 7-12岁 / 其他 |
| 价格与人数 | 原价     | 数字（分） | 是  | 划线价               |
|       | 拼团价    | 数字（分） | 是  | 实际支付价格            |
|       | 成团人数要求 | 数字    | 是  | 达到此人数自动成团         |
| 时间与地点 | 上课开始时间 | 日期时间  | 是  | 课程开始时间            |
|       | 上课结束时间 | 日期时间  | 是  | 课程结束时间            |
|       | 报名截止时间 | 日期时间  | 是  | 默认上课前24小时         |
|       | 上架时间   | 日期时间  | 是  | 必须晚于当前时间          |
|       | 下架时间   | 日期时间  | 否  | 到时间后课程停止拼团        |
|       | 所在区域   | 文本    | 是  | 如“南山区”            |
|       | 详细地点   | 文本    | 是  | 如“中心广场”           |
|       | 经度     | 数字    | 否  | 可根据详细地点自动解析       |
|       | 纬度     | 数字    | 否  | 可根据详细地点自动解析       |
| 教练信息  | 教练姓名   | 文本    | 是  | 显示在课程详情页          |
|       | 教练简介   | 多行文本  | 是  | 资质经验描述            |
|       | 教练证书   | 多图    | 否  | 证书图片，可多张          |
| 课程介绍  | 课程介绍   | 富文本   | 是  | 图文详情              |



* **交互**：

  * 提交后保存课程，返回列表页

  * 新建课程时，所在区域拆分为省 / 市 / 区三级选择，默认仍以所选三级区域拼接保存到“所在区域”字段。

  * 详细地点输入支持地图联想搜索，运营可输入场馆名、小区名或详细地址后，从候选结果中选择目标地点。

  * 选择地点候选并点击“确定位置”后，系统需自动回填经纬度，并自动更新省 / 市 / 区三个字段，保持与地图返回的行政区划一致。

  * 若运营未直接选择候选地点，也可通过“根据详细地点解析坐标”执行地理编码；解析成功后同样需自动回填省 / 市 / 区及经纬度。

* **表单验证**：

  * 拼团价不能大于原价

  * 上架时间必须晚于当前时间

  * 上架时间必须早于报名截止时间

  * 上课开始时间必须在报名截止时间之后

  * 下架时间如填写，必须晚于上架时间

  * 封面图、课程名称、上课时间等必填字段不可为空

  * 课程类别必填，当前仅允许选择“体适能”或“跳绳”



### 3.4 订单管理



#### 3.4.1 订单列表页

* **列表展示**：

  * 表格字段：订单号、用户昵称、课程名称、支付金额（元）、状态（待支付/已支付/已退款）、支付时间、退款时间、退款类型、退款结果、操作（查看详情、手动退款）

  * 支持按订单号、用户昵称、课程名称搜索

  * 支持按订单状态、退款类型、时间范围筛选

  * 时间范围支持按不同维度筛选：
    * 下单时间
    * 支付时间
    * 退款时间

  * 支持分页

* **订单详情**：

  * 点击查看详情，展示完整订单信息（包括用户信息、课程信息、拼团信息、支付信息、退款信息等）

* **手动退款**：

  * 仅对“已支付”的订单可操作

  * 点击后弹出确认框，输入退款原因，确认后执行后台退款流程

  * 退款成功后订单状态变为“已退款”，记录退款时间及原因

  * 当前实现中，手动退款成功后会自动回滚：
    * 该用户参团记录
    * 拼团当前人数
    * 拼团状态

* **自动退款展示**：

  * 当课程因报名截止前无成功团而进入“拼团失败”后，订单列表需自动展示退款状态、退款时间和退款原因。

  * 自动退款类型固定展示为“系统自动退款”。

  * 订单详情中需可区分“系统自动退款”和“手动退款”。



#### 3.4.2 规则

* 当前版本后台订单管理暂不展示用户手机号。

* 手动退款需校验订单状态，防止重复退款。

### 5.1 性能要求

* 页面加载时间 < 2秒

* 列表查询接口响应 < 500ms



### 5.2 安全性

* 所有页面需登录验证，未登录跳转登录页

* 密码加密存储（bcrypt）

* 操作日志已接入关键操作记录，例如：
  * 删除账号
  * 课程创建 / 编辑 / 下架
  * 课程状态同步
  * 课程自动退款
  * 订单手动退款



### 5.3 兼容性

* 支持主流浏览器（Chrome、Edge、Safari最新版）



## 6. 验收标准



### 6.1 登录认证

* 正确用户名密码可登录，错误密码无法登录。

* 未登录状态下无法访问任何后台页面。



### 6.2 账号管理

* 超级管理员可查看、新增、编辑、删除普通管理员账号。

* 普通管理员登录后无账号管理入口。

* 新增账号时密码加密存储。

* 支持按用户名、角色、状态筛选。



### 6.3 课程管理

* 可正常新建、编辑、下架课程，列表展示正确。

* 课程列表支持按课程名称、状态、时间维度与日期范围筛选。

* 课程详情支持只读查看、返回上一级和查看拼团记录。

* 课程状态按以下规则自动流转且展示正确：

  * 上架前为“待上架”

  * 上架后进入“拼团中”

  * 截止前有成功团则进入“等待上课”

  * 开课后进入“上课中”

  * 结束后进入“已结课”

  * 截止前无成功团则进入“拼团失败”

* 拼团失败后，系统会自动触发退款标记，并更新退款时间与退款原因。

* 课程配置的所有字段都能在家长端正确展示（见数据映射）。



### 6.4 拼团管理

* 拼团仍作为课程下子实体存在，但后台提供独立拼团管理模块用于查看和筛选全量拼团。

* 后台提供独立拼团管理模块，同时支持课程维度查看某课程下全部拼团。

* 同一课程同一时刻仅允许一个 `active` 拼团。

* 新建拼团的截止时间统一等于课程报名截止时间。

* 拼团失败后，团状态正确更新为 `failed`，但 `group_members` 历史记录保留。

* 同一用户同一课程的重复参团限制仅按成功团/成功订单判断，不因失败团成员记录被误拦截。

* 拼团详情页可查看成员、订单、退款结果和异常排查提示。



### 6.5 订单管理

* 列表正确展示订单信息、退款信息与分页信息。

* 支持按订单号、昵称、课程名、订单状态、退款类型、时间范围筛选。

* 失败课程触发自动退款后，订单页能正确展示退款状态、退款时间、退款原因和退款类型。

* 订单详情可查看完整退款信息、课程信息和拼团信息。

* 手动退款功能正常，退款后订单状态变更，并回滚拼团人数与拼团状态。
