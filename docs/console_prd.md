## 1. 产品概述



### 1.1 项目背景

运营后台是邻动体适能小程序的内部管理工具，供平台运营人员使用，用于配置课程、管理订单、维护管理员账号。后台所有课程配置数据将直接决定小程序家长端展示的课程信息，因此字段设计需与家长端保持严格一致。



### 1.2 产品定位

一个面向内部运营人员的Web管理后台，用于支撑小程序的日常运营工作，保证课程信息准确、订单处理及时、后台账号安全可控。



### 1.3 核心目标

* 提供课程管理功能，支持课程的新增、编辑、下架，课程字段与家长端展示字段一一对应。

* 提供订单管理功能，支持订单查看、搜索、手动退款，订单字段与家长端订单信息一致。

* 提供账号管理功能，支持后台管理员账号的增删改查。

* 确保后台访问安全，通过登录认证控制权限。



## 2. 功能范围



| 模块       | 功能点   | 说明                        |
| -------- | ----- | ------------------------- |
| **登录认证** | 管理员登录 | 用户名+密码登录，验证身份后获取后台token   |
| **账号管理** | 账号列表  | 展示所有后台管理员账号，支持搜索、新增、编辑、删除 |
| **课程管理** | 课程列表  | 展示所有课程，支持搜索、筛选、新增、编辑、下架   |
| **订单管理** | 订单列表  | 展示所有订单，支持搜索、查看详情、手动退款     |



## 3. 详细功能需求



### 3.1 登录认证



#### 3.1.1 登录页

* **页面元素**：

  * 用户名输入框

  * 密码输入框（掩码）

  * 登录按钮

* **交互**：

  * 输入用户名和密码，点击登录，后端验证。

  * 验证成功：跳转至后台首页（课程管理列表页），存储token。

  * 验证失败：提示“用户名或密码错误”。

* **规则**：

  * 密码需加密传输（HTTPS），后端使用bcrypt存储和比对。

  * token有效期建议7天，过期需重新登录。



### 3.2 账号管理



#### 3.2.1 账号列表页

* **列表展示**：

  * 表格字段：ID、用户名、角色（超级管理员/普通管理员）、最后登录时间、创建时间、操作（编辑、删除）

  * 支持按用户名搜索

  * 支持分页

* **新增账号**：

  * 弹窗表单：用户名、密码、确认密码、角色（下拉选择）

  * 提交后，后端创建账号，密码加密存储

* **编辑账号**：

  * 弹窗表单：用户名（不可修改）、密码（可留空不修改）、角色

  * 提交后更新信息

* **删除账号**：

  * 点击删除，二次确认后执行软删除或物理删除

* **权限说明**：

  * 超级管理员：拥有所有权限（可管理账号、课程、订单）

  * 普通管理员：仅可查看/管理课程和订单，不可管理账号



#### 3.2.2 账号管理规则

* 至少保留一个超级管理员账号，不可删除。

* 新增账号时密码需符合强度要求（至少6位）。



### 3.3 课程管理



#### 3.3.1 课程列表页

* **列表展示**：

  * 表格字段：课程ID、课程名称、上课时间、上课地点、拼团价、原价、成团人数、状态（待开始/进行中/已结束）、操作（编辑、下架）

  * 支持按课程名称、时间范围搜索

  * 支持分页

* **新建课程**：

  * 按钮跳转至新建表单页

* **编辑课程**：

  * 点击编辑进入编辑页，可修改所有字段

* **下架课程**：

  * 点击下架，二次确认后课程状态变为“已结束”，前端不再显示



#### 3.3.2 课程表单页

表单字段需与家长端展示字段一一对应，详见第4章“数据字段映射”。



| 字段分组  | 字段名    | 字段类型  | 必填 | 说明                |
| ----- | ------ | ----- | -- | ----------------- |
| 基本信息  | 课程名称   | 文本    | 是  | 显示在课程卡片、详情页       |

|       | 轮播图    | 多图    | 否  | 课程详情页顶部轮播         |
|       | 适用年龄   | 单选    | 是  | 3-6岁 / 7-12岁 / 其他 |

|       | 拼团价    | 数字（分） | 是  | 实际支付价格            |
|       | 成团人数   | 数字    | 是  | 达到此人数自动成团         |
| 时间与地点 | 上课开始时间 | 日期时间  | 是  | 课程开始时间            |
|       | 上课结束时间 | 日期时间  | 是  | 课程结束时间            |

|       | 详细地点   | 文本    | 是  | 如“中心广场”           |

* **交互**：

  * 提交后保存课程，返回列表页

* **表单验证**：

  * 拼团价不能大于原价

  * 上课开始时间必须在报名截止时间之后

  * 封面图、课程名称、上课时间等必填字段不可为空



### 3.4 订单管理



#### 3.4.1 订单列表页

* **列表展示**：

  * 表格字段：订单号、用户昵称、用户手机号、课程名称、支付金额（元）、状态（已支付/已退款）、下单时间、操作（查看详情、手动退款）

  * 支持按订单号、用户昵称、手机号、课程名称搜索

  * 支持分页

* **订单详情**：

  * 点击查看详情，弹窗或跳转页展示完整订单信息（包括用户信息、课程信息、拼团信息、支付信息等）

* **手动退款**：

  * 仅对“已支付”的订单可操作

  * 点击后弹出确认框，输入退款原因，确认后调用微信退款接口

  * 退款成功后订单状态变为“已退款”，记录退款时间及原因



#### 3.4.2 规则

* 用户手机号来自用户授权获取，需展示完整号码（便于联系）。

* 手动退款需校验订单状态，防止重复退款。

## 5. 非功能性需求



### 5.1 性能要求

* 页面加载时间 < 2秒

* 列表查询接口响应 < 500ms



### 5.2 安全性

* 所有页面需登录验证，未登录跳转登录页

* 密码加密存储（bcrypt）

* 操作日志（可选，记录关键操作，如删除账号、手动退款）



### 5.3 兼容性

* 支持主流浏览器（Chrome、Edge、Safari最新版）



## 6. 验收标准



### 6.1 登录认证

* 正确用户名密码可登录，错误密码无法登录。

* 未登录状态下无法访问任何后台页面。



### 6.2 账号管理

* 超级管理员可查看、新增、编辑、删除普通管理员账号。

* 普通管理员登录后无账号管理入口。

* 新增账号时密码加密存储。



### 6.3 课程管理

* 可正常新建、编辑、下架课程，列表展示正确。

* 课程状态随上课时间自动变化（由定时任务处理，后台可手动刷新）。

* 课程配置的所有字段都能在家长端正确展示（见数据映射）。



### 6.4 订单管理

* 列表正确展示订单信息，包含用户手机号。

* 支持按订单号、昵称、手机号、课程名搜索。

* 手动退款功能正常，退款后订单状态变更，用户收到退款通知。
